### Overview  
This is the official GitHub organization for scrum team **Bear Down** for **CSPSC 44000 – Software Engineering**. 

This organization hosts and manages source code repositories for Bear Down group projects completed for our Software Engineering course.

The primary goal of this organization is to provide a centralized location for our course-related project repositories, collaborate using Agile development practices and Scrum-based development, and enforce professional development workflows.

## Course Coding Standards  

- All code must compile and execute without errors or warnings  
- Plagiarism in any form is not acceptable  
- Any non-original work must be appropriately identified and credited in your `README.md` file  
- Use of ChatGPT and other AI must be detailed at the top of each source code file  
- Only source files required to compile and execute an application plus `README.md` and `LICENSE` files should be submitted  
- The `README.md` file should include your name, the project name, and credits for all non-original content  
- The `README.md` file should include instructions for compiling and executing the application  
- A `LICENSE` file should be included and must be MIT, GNU General Public License (GPL), or all rights reserved  
- Lines should be consistently wrapped at 100 or 120 characters  
- Tabs should be utilized for line indentation (not spaces) and only line feeds utilized (LF, `/n`) at the end of lines  
- Submitted zip files should extract all files into the current folder and not create subfolders  
- All variable, function, and class naming should be descriptive and follow CamelCase naming conventions with classes starting with a capital letter and everything else starting with lower case letters  
- Source code must be commented, with comments being valuable but conservative  
- When classes are utilized, each class should be in a separate source file  
- File names should be meaningful and consistently follow either a CamelCase or dash-case naming convention  
- Source code quotes should be single straight quotes when possible, and user-visible quotes should be curly  
- All files should be UTF-8 compatible text files  
- Git and GitHub should be utilized to manage code  
- Each completed User Story should be associated with a GitHub commit message that identifies the story identifier
